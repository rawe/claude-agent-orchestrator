## Key Architecture Decisions

1. **SSE `session_updated` is authoritative** for status — `run_completed` events do NOT override status
2. **Only idle sessions can be resumed** — terminal sessions (finished/stopped/failed) start new chats
3. **Reaper joins through runs table** — sessions have no `runner_id` column, reaper finds ownership via most recent run
4. **Reaper grace period** — 2 minutes after coordinator startup before reaping (allows runners to reconnect)
5. **Reaper thresholds configurable** via env vars for testing and tuning
