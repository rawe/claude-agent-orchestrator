# Slide: Autonomous vs Procedural Agents

## Title
**"AI for Thinking. Code for Doing."**

## Key Message
Some tasks need AI creativity. Others just need reliable execution. Use the right tool for each job - and switch between them anytime.

## Visual Description

**Layout:** Side-by-side comparison with clear contrast

**Design:**
```
┌─────────────────────────────┐    ┌─────────────────────────────┐
│      AUTONOMOUS             │    │      PROCEDURAL             │
│      [brain icon]           │    │      [checklist icon]       │
│                             │    │                             │
```

**Visual Style:**
- Two equal-sized panels, side by side
- Left panel: Purple background (#9B59B6) with white text
- Right panel: Orange background (#E67E22) with white text
- Speech bubbles show example tasks
- Simple icons at top (brain vs checklist)

**Key visual element:** A **slider or toggle** at the bottom showing these can be swapped:

```
     AUTONOMOUS ◀────●────▶ PROCEDURAL
              "Same interface"
```

## Text on Slide

**Left Panel - Autonomous:**
- Header: "Autonomous (AI)"
- Subhead: "Thinks and decides"
- Example: "Analyze this document..."
- Traits: Creative, Contextual, Adaptive
- Best for: Research, Analysis, Writing

**Right Panel - Procedural:**
- Header: "Procedural (Code)"
- Subhead: "Follows instructions"
- Example: "Get X, transform, save to Y"
- Traits: Fast, Predictable, Cheap
- Best for: Data tasks, File ops, Integrations

**Bottom callout:**
"Start with AI. Replace with code when patterns emerge."

## Why This Visualization Works

1. **Side-by-side = clear comparison** - no ambiguity about differences
2. **Equal panel sizes** - neither is "better", just different
3. **Color coding matches Phase 3 slide** - purple=AI, orange=code (consistent)
4. **Real example tasks** - not abstract descriptions
5. **"Best for" lists** - immediately answers "when do I use which?"
6. **The slider visual** - shows they're interchangeable, not locked in
7. **No technical jargon** - "Thinks and decides" vs "Follows instructions"

## Presenter Notes
Connect this to the previous slide: "Remember Phase 3? This is how you decide which parts to convert. If a task is repetitive and predictable, make it procedural. If it needs judgment, keep it autonomous."
