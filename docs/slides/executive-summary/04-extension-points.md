# Slide: The 4 Extension Points

## Title
**"4 Ways to Customize - Pick What You Need"**

## Key Message
The framework grows with you. Four clear places where you can add, change, or extend - without touching the core system.

## Visual Description

**Layout:** Central hub with 4 spokes radiating outward (compass/wheel pattern)

**Design:**
```
                    ┌─────────────────┐
                    │ AGENT           │
                    │ DEFINITIONS     │
                    │ [doc icon]      │
                    │ "PMs can write" │
                    └────────┬────────┘
                             │
    ┌─────────────┐          │          ┌─────────────┐
    │ CUSTOM      │          │          │ EXECUTORS   │
    │ APPS        │──────────●──────────│             │
    │ [screen]    │    COORDINATOR      │ [gear icon] │
    │ "Your UI"   │                     │ "AI or code"│
    └─────────────┘          │          └─────────────┘
                             │
                    ┌────────┴────────┐
                    │ MCP SERVERS     │
                    │ [plug icon]     │
                    │ "Connect tools" │
                    └─────────────────┘
```

**Visual Style:**
- Center: Small green circle with "CORE" label (represents the stable center)
- 4 boxes positioned at compass points (N, S, E, W)
- Each box has:
  - Bold title
  - Simple icon
  - 2-3 word benefit description
- Connecting lines are dashed (showing "pluggable" nature)

**Color scheme:**
- Center (Core): Green (#2ECC71) - stable
- Agent Definitions: Blue (#3498DB) - content/creative
- Executors: Orange (#E67E22) - power/action
- MCP Servers: Purple (#9B59B6) - connections
- Custom Apps: Teal (#1ABC9C) - user-facing

**Icons (simple line icons):**
- Agent Definitions: Document/notepad
- Executors: Gear/cog
- MCP Servers: Plug/connector
- Custom Apps: Screen/window

## Text on Slide

**Each box contains only:**

| Extension Point | Tag Line | Who |
|-----------------|----------|-----|
| Agent Definitions | "What agents do" | PMs, Concept people |
| Executors | "How agents work" | Developers |
| MCP Servers | "What agents connect to" | Integrators |
| Custom Apps | "What users see" | UI/UX team |

**Small footer:** "Core stays stable. Extensions grow independently."

## Why This Visualization Works

1. **Compass/wheel metaphor** - immediately shows "multiple entry points"
2. **Center stability** - the green core shows "this doesn't change"
3. **Equal sizing** - no extension point is more important than others
4. **Dashed lines** - visual cue for "pluggable/optional"
5. **Who tags** - instantly answers "is this for me?" for different roles in the room
6. **No technical terms** - "Agent Definitions" not "JSON configs", "Connect tools" not "API adapters"
